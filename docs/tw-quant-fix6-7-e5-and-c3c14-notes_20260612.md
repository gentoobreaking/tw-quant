## 修改 6：E5 量縮出場信號

### 問題
主力退場前常出現「價穩量縮」——股價還在月線上，但成交量連續萎縮。現有 E1~E4 都沒抓到這個信號。

### 新增 E5 邏輯
- 連續 3 日成交量 < 5 日均量 × 60%
- 且收盤價仍在 MA20 之上
- → 列為次要出場信號「主力退場前兆」

### 參數
| 參數 | 預設值 | 說明 |
|------|--------|------|
| `volume_shrink_ratio_e5` | 0.6 | 量縮門檻（成交量 < 5MA × 60%） |
| `consecutive_shrink_e5` | 3 | 連續量縮天數 |

### 改動檔案
| 檔案 | 改動 |
|------|------|
| `common/scoring.py` | `DEFAULT_EXIT_PARAMS` 新增 E5 參數 |
| `common/scoring.py` | `check_exit` 新增 E5 判斷邏輯 |

### 安全設計
- E5 參數用 `params.get(key, default)` 讀取，自定義 params 不含 E5 key 也不報錯
- 需同時滿足「連續量縮」+「價在月線上」才觸發，避免下跌趨勢中誤報

### 測試結果
- pytest 43/43 ✅
- 連3日量縮33%~56% + 價穩在MA20上 → E5 觸發 ✅
- 只有2天量縮 → 不觸發 ✅
- 量縮但股價在MA20下 → 不觸發 ✅
- 量縮比例66% > 60%門檻 → 不觸發 ✅

## 修改 7：C14 vs C3 差異註解

### 問題
C3（回測20MA）和 C14（距60MA proximity）容易混淆，前者是「低價碰月線」，後者是「收盤距季線百分比」。

### 註解內容（stock_screener.py + etf_screener.py）

**C3 註解：**
```
# C3: 回測 20MA（近 N 日低價曾觸及月線）
# 與 C14 的差異：C3 是「低價碰月線」= 回測站回，C14 是「收盤距季線 X%」= 位階判斷
# C3 用低價觸及、lookback 短期（5日），C14 用收盤距離、MA60 長期（60日）
```

**C14 註解：**
```
# C14: 距60MA 位階（0–5% 範圍，非「回測站回」）
# 與 C3 的差異：C3 是「低價碰 20MA」= 短期回測買點；
# C14 是「收盤距 60MA 的百分比」= 中期位階判斷，proximity 設計更穩健
# 兩者互補：C3 抓短期支撐，C14 抓中期偏離度
```
